对泛型的一些理解

发表于2022-05-17|更新于2022-05-18|八股文
|字数总计:344|阅读时长:1分钟|阅读量:
泛型

泛型本质是将数据类型参数化，它通过擦除的方式实现。声明了泛型.java源代码，在编译生成.class文件之后，泛型相关的信息就消失了。可以认为，源代码中泛型的信息是提供给编译器用的，对JAVA虚拟机不可见。

看下面的代码:

CODE
public static void main(String[] args) {
       List<String> a = new ArrayList<>();
       List<Integer> b = new ArrayList<>();
       System.out.println(a.getClass()==b.getClass());
}

打印的结果是true，也证明了在字节码中并没有保存泛型信息。泛型中类型的作用更多是在编译器进行语法检查，和对返回的对象等进行强转。

通过反射获取泛型类型
public class test {
    public Map<Integer,String> map = new HashMap<>();
    public static void main(String[] args) throws NoSuchFieldException {
         Field fa = test.class.getField("map");
        System.out.println(fa.getType());
        Type type = fa.getGenericType();
        if(type instanceof ParameterizedType){
            ParameterizedType pt= (ParameterizedType) type;
            Type[] t= pt.getActualTypeArguments();
            for (Type type1 : t) {
                System.out.println(type1);
            }
        }
    }
}

getType(): 返回一个对象，它标识了此Field对象所表示字段的声明类型。

getGenericType(): 返回一个 Type 对象，它表示此 Field 对象所表示字段的声明类型。

getGenericType()的源码

public Type getGenericType() {
  return (Type)(this.getGenericSignature() != null ? this.getGenericInfo().getGenericType() : this.getType());
}

如果 getGenericSignature 不为空，即如果当前类型是泛型，则返回泛型类型，反之返回 getType() 的值。
